# UI Component Classification and Visual Layout Analysis for Webpages

Machine learning pipeline for detecting UI components in webpage snapshots using YOLOv11s. Evaluating detected components with standard mathematical approach for clutter, alignment and color contrast.

## Setup

### 1. Clone and Install

```bash
git clone <repository-url>
cd component-classifier-backend/Python/
pip install -r requirements.txt
```

### 2. Set Python Path

**Important:** Set PYTHONPATH to ensure imports work correctly.

#### Windows (PowerShell)

```powershell
cd component-classifier-backend/Python/
$env:PYTHONPATH = $PWD

# Verify
echo $env:PYTHONPATH
```

#### macOS / Linux (Bash)

```bash
cd component-classifier-backend/Python/
export PYTHONPATH=$PWD

# Verify
echo $PYTHONPATH
```

**Expected output:** You should see the full path to your `Python/` directory.

> **Note:** This must be set each time you open a new terminal. To make it permanent, add the export/set command to your shell profile (`.bashrc`, `.zshrc`, or PowerShell profile).

---

### 3. Prepare Dataset

```bash
mkdir -p data/raw_data/
```

Download webui-70k dataset and extract into `data/raw_data/`

### 4. Configure

```bash
cp config/config_example.yml config/config.yml
```

Edit `config.yml` with your settings

---

## Run Pipeline

### Step 1: Directory Cleanup

```bash
python data_preprocessing\directory_cleanup.py
```

**Output:** Unwanted files removed from `data/raw_data/`. Folders with incorrect file counts deleted and logged to `txt/deleted_folder.txt`

---

### Step 2: Initialize Resource

```bash
python data_preprocessing\resource_initializer.py
```

**Output:** Images moved to `datasets/images/`, `.gz` files extracted to `.json` in place. Failed image copies logged to `txt/failed_images.txt`

---

### Step 3: Preprocess Data

```bash
python data_preprocessing\json_processor.py
```

**Output:** YOLO label `.txt` files created in `datasets/labels/`, one per image. Image-label pairs guaranteed to be in sync.

---

### Step 4: Split Data

```bash
python training\data_splitter.py
```

**Output:** `split_data/train/`, `split_data/validation/`, `split_data/test/` populated at 80/10/10 ratio with reproducible seed.

---

### Step 5: Train Model

```bash
python training\train.py
```

**Output:** `checkpoints/train/weights/best.pt` and `last.pt`. Mid-training callbacks automatically write `evaluation_results/iteration_losses_TIMESTAMP.csv` and `evaluation_results/epoch_per_class_metrics_TIMESTAMP.csv` throughout training.

---

### Step 6: Evaluate Model

```bash
python evaluation\evaluator.py
```

**Output:** `evaluation_results/per_class_metrics_TIMESTAMP.csv` and `evaluation_results/overall_metrics_TIMESTAMP.csv` with precision, recall, F1, mAP50, mAP50-95 per class. `evaluation_results/test_evaluation/` folder with YOLO-generated confusion matrix plots. Results also printed as a formatted table in the terminal.

---

### Step 7: Generate Graphs

```bash
python evaluation\visualizer.py
```

**Output:** 7 graphs saved to `evaluation_results/graphs/` — per-class F1 bar, precision vs recall comparison, per-class mAP50 bar, iteration-level loss curves, per-class F1 progression across epochs, overall mAP50 progression, and train vs validation loss comparison for overfitting detection.

---

### Step 8: Run Analysis Pipeline

```bash
python inference\predictor.py [image-path]
```

**Output:** `analysis_results/layout_report_TIMESTAMP.json` with complete detection and analysis metadata.

## Resume Training

Edit `training/config.py`:

```python
TRAINING_CONFIG = {
    "model": "checkpoints/train/weights/last.pt",
    "epochs": 750,
}
```

Edit `training/train.py`:

```python
results = model.train(
    ...
    resume=True,
)
```

Then run:

```bash
python training\train.py
```
